# 校园地图导航系统

## 一、项目概述

本项目是一个基于 Vue 3 + Pinia + Element Plus 的校园地图导航系统，提供地图浏览、POI 管理、路线规划、导航模拟等功能。

### 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Vue | 3.x | 前端框架 |
| Vue Router | 4.x | 路由管理 |
| Pinia | 3.x | 状态管理 |
| Element Plus | 2.x | UI 组件库 |
| Axios | 1.x | HTTP 客户端 |
| Vite | 5.x | 构建工具 |
| ECharts | 6.x | 图表库 |

### 核心功能

1. **地图浏览**：支持缩放、拖拽、坐标显示
2. **POI 管理**：增删改查各类校园点位
3. **路线规划**：基于路网的最短路径计算
4. **导航模拟**：路径可视化展示
5. **权限管理**：用户登录、角色控制

## 三、核心模块

### 3.1 地图模块

**功能**：地图缩放、拖拽、POI 渲染、路线绘制

**关键组件**：
- `MapBackground.vue` - 地图背景容器
- `PoiCanvas.vue` - POI Canvas 渲染层
- `RouteCanvas.vue` - 路线 Canvas 渲染层

**组合式函数**：
- `useMapTransform.js` - 地图变换逻辑
- `useMapEvents.js` - 地图事件处理
- `usePoiHandlers.js` - POI 事件处理

### 3.2 POI 管理模块

**功能**：POI 的增删改查、类型筛选、显隐控制

**POI 类型**：
| 类型 | 说明 |
|------|------|
| dormitory | 宿舍 |
| classroom | 教学楼 |
| entrance | 进出口 |
| canteen | 食堂 |
| statue | 雕像 |
| clinic | 医务室 |
| supermarket | 超市 |
| parking | 停车场 |
| toilet | 卫生间 |
| stadium | 体育场 |
| waypoint | 路径点 |
| other | 其他 |

### 3.3 路网与导航模块

**功能**：路径点管理、路网连接、最短路径计算

**数据模型**：
- **路径点（Waypoint）**：地图上的导航节点
- **连接（Connection）**：路径点之间的道路连接
- **路线（Route）**：预定义的游览路线

### 3.4 用户权限模块

**功能**：用户登录、角色认证、权限控制

**角色等级**：
| 角色 | 权限 |
|------|------|
| 未登录 | 仅可访问登录页和错误页面 |
| 普通用户 | 可浏览地图、搜索、评论 |
| 管理员 | 可管理 POI、路线、用户 |

## 五、状态管理

### 5.1 Store 列表

| Store | 职责 |
|-------|------|
| `poiStore` | POI 数据管理 |
| `mapStore` | 地图全局状态（缩放、显隐控制） |
| `routeStore` | 路线数据管理 |
| `roadNetworkStore` | 路网连接数据 |
| `navigationStore` | 导航状态（起点终点、路径） |
| `userStore` | 用户认证状态 |
| `iconStore` | 图标配置 |

### 5.2 核心状态

**mapStore 关键状态**：
- `centerOnPoi` - POI 定位函数（全局注册）
- `currentScale` - 当前缩放比例
- `showPois` / `showRoutes` / `showNetwork` - 显隐开关
- `visiblePoiTypes` - POI 类型显隐配置

## 六、开发与构建

### 6.1 命令

```bash
# 安装依赖
npm install

# 开发模式
npm run dev

# 构建生产版本
npm run build

# 预览构建结果
npm run preview
```

### 6.2 环境配置

- **Base URL**：`/api`（可在 `utils/request.js` 中修改）
- **端口**：默认 5173（Vite 默认端口）
- **代理配置**：可在 `vite.config.js` 中配置

---

## 七、部署说明

1. 确保后端 API 服务已启动
2. 配置 `vite.config.js` 的代理目标
3. 构建生产版本：`npm run build`
4. 部署 `dist` 目录到静态服务器

## 八、功能亮点

1. **Canvas 渲染**：高性能地图渲染，支持大量 POI 显示
2. **全局状态管理**：通过 Pinia 实现状态共享
3. **权限控制**：基于角色的访问控制
4. **路网导航**：支持最短路径计算和可视化
5. **响应式设计**：适配不同屏幕尺寸
