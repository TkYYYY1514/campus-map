## 三、路由守卫说明

### 3.1 全局守卫逻辑

1. **等待认证加载**：若 `userStore.isAuthLoading` 为 true，等待加载完成
2. **错误页面放行**：`/401`、`/403`、`/404` 直接放行
3. **登录页处理**：已登录用户访问 `/login` 自动跳转到 `/map`
4. **认证检查**：`meta.requiresAuth: true` 的路由需要登录
5. **权限检查**：`meta.adminOnly: true` 的路由需要管理员权限

### 3.2 权限等级

| 等级 | 说明 | 可访问页面 |
|------|------|----------|
| 未登录 | 匿名用户 | `/login`、错误页面 |
| 普通用户 | 已登录 | 除管理员页面外的所有页面 |
| 管理员 | 已登录且角色为 admin | 所有页面 |

## 四、命名约定

### 4.1 路由名称
- 使用 PascalCase（大驼峰）命名
- 见名知意，如 `RouteManagement`、`PoiOperation`

### 4.2 路径命名
- 使用 kebab-case（短横线分隔）
- 小写字母，如 `/route-management`、`/nav-sim`

### 4.3 文件命名
- 组件文件：PascalCase（如 `CampusMapInfo.vue`）
- 工具函数：camelCase（如 `useMapTransform.js`）
- 目录名称：kebab-case（如 `route-management`）
